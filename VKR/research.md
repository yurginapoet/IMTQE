**Что ищем одновременно:**

Гиперпараметры каждой модели независимо, потом веса ансамбля поверх. Это двухуровневая задача. Модели можно оптимизировать параллельно, веса ансамбля — только после того как модели обучены.

**Конкретно для каждой модели:**

XGBoost — `learning_rate`, `max_depth`, `min_child_weight`, `subsample`, `colsample_bytree`, `reg_lambda`, `reg_alpha`, `gamma`. Сейчас всё захардкожено в `train_sentence_model.py`. Пространство большое, здесь хорошо работает Bayesian optimization (например, `optuna`).

Ridge — только `alpha`. Тут достаточно grid search по логарифмической шкале.

Random Forest — `n_estimators`, `max_depth`, `min_samples_leaf`, `max_features`. Пространство среднее, тоже Bayesian.

**Веса ансамбля:**

После того как три модели обучены с лучшими гиперпараметрами, веса ансамбля ищутся отдельно. Это можно сделать через `scipy.optimize.minimize` с ограничением `w1 + w2 + w3 = 1, wi >= 0`, оптимизируя Pearson r на val-сплите. Или перебором по сетке — пространство маленькое (три переменные с суммой 1), это недорого.

**Метрика оптимизации:**

На val-сплите — Pearson r (как сейчас в early stopping XGBoost). Финальная проверка — Spearman ρ на MQM external test. Важно не оптимизировать напрямую по MQM test, иначе переподгонка.

**Что параллелить реально:**

Поиск гиперпараметров для XGBoost, Ridge и RF — полностью независимы, запускаются одновременно. Optuna поддерживает параллельные trials через `n_jobs` или отдельные процессы с общим storage. Поиск весов ансамбля — последовательно после всех трёх.

**Риски:**

Главный — переоптимизация под val-сплит. Если val небольшой (у нас 10% от DA), то Pearson r на нём нестабилен. Лучше использовать k-fold cross-validation внутри train-сплита для оценки гиперпараметров, а val оставить только для финальной проверки весов ансамбля.

Второй риск — время. XGBoost с 4000 деревьями на каждый trial медленный. Нужно либо ограничить `num_boost_round` во время поиска (например, 1000), либо использовать `XGBRegressor` с `early_stopping_rounds` внутри optuna callback.
